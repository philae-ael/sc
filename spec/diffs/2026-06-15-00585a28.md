## Diff

Update `sc clone <url> [targetDir]` behavior: after cloning the repository bare into `<targetDir>/.git`, detect the initial branch to check out as a worktree. Prefer the repository default branch when it is `main` or `master`; otherwise prefer an existing `main` branch, then an existing `master` branch, then fall back to the default branch. Create the worktree using the selected branch name as both the worktree directory name under the SC root and the checked-out branch/ref, rather than always creating `<targetDir>/main` from `HEAD`. Update CLI surface/spec wording and implementation helper list as appropriate.

## Context

User requested: "the clone subcommand should auto add a worktree for the main / master branch". Current implementation always creates `<dir>/main` from `HEAD`.
