Update the `sc commit` command specification to add optional `--amend` flag support:

- `sc commit <message>` / `sc c <message> [--amend]`
  - When `--amend` is provided, amends the last commit with the new message.
  - When `--amend` is not provided:
    - Stages all changes with `git add -A` when changes exist.
    - Counts consecutive `[SC-STAGE]` commits from `HEAD`.
    - Soft-resets before those staging commits so they are squashed into the new normal commit.
    - Creates a normal commit with the provided message.
  - Prints `No changes to commit` when the worktree has no changes.
