# Fix --amend flag to integrate with staging squashing

The `--amend` flag was not staging changes or handling SC-Stage commits before amending. This is now fixed.

Update the `sc commit` command specification:

- `sc commit <message>` / `sc c <message> [--amend]`
  - Stages all changes with `git add -A` when changes exist.
  - Counts consecutive `[SC-STAGE]` commits from `HEAD`.
  - Soft-resets before those staging commits so they are squashed into the commit.
  - When `--amend` is provided: amends the resulting HEAD with the new message.
  - When `--amend` is not provided: creates a new commit with the new message.
  - Prints `No changes to commit` when the worktree has no changes and `--amend` is not used.

Previously, `--amend` bypassed staging and squashing logic, leaving SC-Stage commits in place. The corrected behavior ensures SC-Stage commits are always squashed before amending, matching the normal commit flow.
